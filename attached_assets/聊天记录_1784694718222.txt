方向 B（大胆）：编辑器本身做成什么样？
这是最有意思的设计问题。有几种完全不同的方向：

B1 — 普通代码编辑器 BasicTextField + VisualTransformation 做语法高亮，行号，滚动。实用，但没有灵魂，跟 VS Code Mobile 没什么区别。

B2 — Command bar 即编辑器 极端路线：进入文件后，命令栏仍然是主角。所有操作（跳转符号、搜索替换、执行代码块）都通过它完成，编辑区域只是"结果视图"。类似 Helix/Kakoune 的哲学移植到触控上——手机从来没有过真正 modal 的编辑器，这会是 Axiom 的差异点。

B3 — 排版优先的写作编辑器 不是 IDE，是给写作用的。每个项目可以选字体/间距/行宽，进入项目就是沉浸式全屏排版面。主页那种克制美学延伸进去。目标用户是写作者、笔记者，不是程序员。

方向 C（大胆）：主页动效 → 项目过渡动画
现在打开项目是 Navigator.navigate()，直接切屏。但整个主页花了这么多精力做 spring 动效，进入项目的瞬间是浪费掉的。大胆做法：项目卡片点击后做 shared element transition——卡片展开成编辑器界面，命令栏从主页"继承"过来，不是跳页而是变形。Compose 1.7+ 的 SharedTransitionLayout 已经支持这个。视觉上会非常有冲击力。
